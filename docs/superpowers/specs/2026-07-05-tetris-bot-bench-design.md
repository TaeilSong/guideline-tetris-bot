# 가이드라인 테트리스 + 휴리스틱 봇 + 벤치마크 설계

작성일: 2026-07-05

## 목표

표준 테트리스 가이드라인 규칙을 지키는 웹앱 테트리스를 만들고, 스스로 플레이하는 휴리스틱 봇을 붙이고, 봇의 실력을 시드 고정으로 완전 재현 가능하게 측정하는 CLI를 제공한다.

## 성공 기준

- 브라우저에서 사람이 플레이 가능하고, 봇 자동플레이로 전환 가능하다.
- SRS 회전(월킥), 7-bag, Hold, Next 5개, Ghost, Lock delay, T-스핀 포함 점수 계산이 가이드라인대로 동작한다.
- `node bench/bench.js --games N --seed S`가 같은 인자에 대해 항상 동일한 통계를 낸다.
- core 유닛 테스트(월킥/T-스핀/라인클리어/점수/RNG 재현성)가 통과한다.

## 아키텍처 — 코어/UI/벤치 3계층

```
tetris/
├── src/core/          순수 로직 (브라우저·Node 공용, DOM/시간/전역 랜덤 의존 없음)
│   ├── rng.js         시드 고정 RNG (mulberry32) + 7-bag 생성기
│   ├── pieces.js      SRS 조각 정의, 회전 상태, 월킥 오프셋 테이블
│   ├── board.js       보드 상태, 충돌 검사, 라인 클리어
│   ├── engine.js      게임 상태 머신 (스폰/이동/회전/락딜레이/hold/next/gravity)
│   ├── scoring.js     점수·레벨·콤보·B2B·T-스핀 판정
│   └── bot.js         휴리스틱 평가함수 + 착수 후보 탐색
├── web/
│   ├── index.html
│   ├── render.js      Canvas 렌더 (보드/고스트/hold/next/HUD)
│   └── input.js       키보드 입력 + 봇 자동플레이 토글
└── bench/
    └── bench.js       Node CLI: N판 시뮬레이션 → 통계 표 + JSON
```

핵심 원칙은 core가 DOM·실시간 시계·전역 랜덤에 의존하지 않는 것이다. RNG는 시드를 주입받고, 엔진의 진행은 외부에서 호출하는 틱/액션으로만 이루어진다. 그래야 브라우저에선 60fps로, 벤치에선 최대 속도로 동일 코드가 돈다.

- 모듈은 ES 모듈(`export`/`import`). 브라우저는 `<script type="module">`, Node는 native ESM으로 로드하므로 빌드 도구가 필요 없다.

## 가이드라인 규칙

- SRS 회전: 4회전 상태(0/R/2/L) + JLSTZ 공용 월킥 테이블과 I 전용 테이블. 회전 시 5개 오프셋을 순서대로 시도.
- T-스핀: 마지막 성공 액션이 회전이고 T조각의 4개 코너 중 3개 이상이 막혀 있으면 T-스핀. 앞쪽 두 코너 상태로 T-Spin/T-Spin Mini 구분.
- 7-bag: 7종을 시드 셔플로 한 봉지씩 공급.
- Hold: 조각당 1회 스왑 제한. Next 5개 표시. Ghost 조각 표시.
- Lock delay: 접지 후 0.5s, 이동/회전 시 리셋, 리셋 최대 15회 후 강제 락.
- 점수: Single/Double/Triple/Tetris, T-Spin/Mini 계열 보너스, Soft/Hard drop 가산, Combo, Back-to-Back(테트리스·T-스핀 연속). 레벨업 시 중력 증가는 표준 곡선(가이드라인 프레임 표)을 사용.

## 봇 — 휴리스틱 평가

현재 조각의 모든 (회전 × 가로위치) 조합을 board 복제본에 하드드롭 시뮬레이션하고, 각 결과 보드를 가중합으로 점수화해 최선의 착수를 고른다.

- 지표: aggregate height, 완성 라인 수, holes, bumpiness, 최대 높이, 우물 깊이 등 (Dellacherie/EI-Tetris 계열).
- 가중치는 `bot.js` 상단 상수로 노출해 튜닝·실험 가능하게 한다.
- Hold도 후보에 포함: 현재 조각과 hold 조각 중 더 나은 착수를 선택.
- 봇은 목표 착수(회전 수 + 목표 x + hold 여부)를 반환하고, 브라우저에서는 이를 입력 액션으로 풀어 눈으로 보이게 재생, 벤치에서는 즉시 적용한다.

## 벤치마크 — 재현성

- 실행: `node bench/bench.js --games 100 --seed 42 [--maxPieces 5000] [--json out.json]`
- 각 판은 `seed`에서 파생된 게임별 시드(`derive(seed, gameIndex)`)를 사용해 조각 순서가 완전 고정된다. 같은 인자면 항상 같은 결과.
- 종료 조건: 봇 게임오버 또는 maxPieces 도달. 헤드리스로 core 로직만 고속 실행(렌더/타이머 없음).
- 출력: 라인 수·점수·착수 조각 수의 평균/중앙값/표준편차/최소/최대, 게임오버 사유 분포. 콘솔 표 + `--json` 지정 시 각 판 원자료 저장.

## 테스트

Node 내장 `node:test` + `node:assert`로 core 유닛 테스트.

- RNG: 같은 시드 → 동일 난수/조각 수열.
- 월킥: 대표 킥 시나리오(예: 벽 옆 I/T 회전) 성공/실패.
- T-스핀: 3-corner 판정, Mini 구분.
- 라인 클리어: 다중 라인, 위 블록 낙하.
- 점수: 라인별/드롭/콤보/B2B/T-스핀 가산.
- 봇/벤치: 시드 고정 스냅샷으로 회귀 방지(같은 시드 → 같은 라인 수).

## 범위 밖 (YAGNI)

- 멀티플레이·가비지 라인·파이널 대전.
- 봇 학습(강화학습/유전 알고리즘) — 가중치는 수동 상수로 시작.
- 사운드·모바일 터치 컨트롤.
